密码学基础概念总结

发表于2023-09-18|更新于2023-09-18|密码学

密码学基础概念总结

1. 隐患特性：密码学是为了解决什么问题

1.1 机密性

  • 解决手段：

    1.对称密码
    2.公钥密码

1.2 完整性

  • 概念
    文件是不是真的（是否有替换或缺失）。

  • 解决手段

    1.消息认证码
    2.数字签名
    3.单项散列函数

1.3 认证性

  • 解决手段

    1.数字签名
    2.消息认证码

1.4 不可否认性

  • 解决手段

    数字签名

2. 密码技术：密码学用什么手段解决问题

2.1 对称密码

2.1.1 特点：速度快，但保密性差。

2.1.2 实现方式

1.XOR
2.一次性密码本
3.DES

Feistel network

由多轮组成，例如3轮

4.三重DES
用三个不同的DES密钥，进行DES加密、DES解密、DES加密。
5.Rijndael算法

SPN结构

4个步骤

2.1.3 AES

对称密码标准

Rijndael算法

2.1.4 输入输出

输入明文，输出密文。

2.2 公钥密码

2.2.1 特点

速度慢，但保密性好。

2.2.2 密钥配送问题

方法：

1.事先共享密钥

不现实

2.通过密钥分配中心

有局限性，密钥中心负荷大，密钥中心被黑

3.通过Diffie-Hellman密钥交换

通过公钥密码

只有拥有解密密钥的人才能够解密

2.2.3 公钥密码

加密密码

公开

解密密码

私有

机密性随着密钥长度而变化

2.2.4 分类

  • RSA算法

    密文 = 明文^E mod N

    E和N的组合就是公钥

    明文 = 密文^D mod N

    D和N的组合就是私钥

  • ElGamal方法
  • Rabin方法
  • 圆锥曲线密码

2.2.5 输入输出

输入明文，输出密文

2.3 混合密码系统

2.3.1 特点

输入输出皆具二者优点。

2.3.2 具体实现

用对称密码提高速度，用公钥密码保护会话密钥。

    通过对称密码来加密，将加密消息时使用的密钥通过公钥密码来加密

一句话说，就是用公钥密钥加密对称密钥。

2.4 单项散列函数

2.4.1 特点 输入输出

输入称为消息，输出称为散列值。

2.4.2 性质

  • 1.根据任意长度的消息计算出固定长度的散列值
  • 2.快速计算出散列值
  • 3.消息不同，散列值也不同

    抗碰撞性：难以发现碰撞

  • 4.具备单向性

    根据散列值无法求出消息

2.4.3 标准

SHA-3

新标准的单项散列函数算法

Keccak

内部结构：海绵结构

2.4.4 优缺点

可以识别出篡改；
不可以识别出伪装。

2.5 消息认证码（MAC：Message Authentication Code）

2.5.1 输入输出

    输入
        包括任意长度的消息和一个发送者与接收者之间共享的密钥

    输出
        固定长度的数据，称为MAC值

2.5.2 使用步骤

A.发送者把计算出的MAC值一并发给接收者
B.接收者把共享密钥和消息认证码组合，计算出MAC值
C.比对二者的MAC值

2.5.3 实现方法

  • 使用单项散列函数实现
    HMAC

  • 使用分组密码实现
    CBC模式

2.5.4 缺点

  • 对第三方的证明
  • 无法防止否认

2.5.5 攻击方式

重放攻击

实现保存正确的MAC值，然后不断发送

解决方法

序号
时间戳

密钥推测攻击

2.6 数字签名

2.6.1 步骤与流程

  • 步骤

    生成消息签名

    签名密钥只能由签名的人持有

    验证消息签名

    任何需要验证签名的人都可以拥有

  • 流程

    消息经过私钥加密，生成密文（数字签名），用公钥解密得到消息（验证签名）

2.6.2 实现方法

对散列值签名
对消息直接签名

2.6.3 攻击方式

  • 中间人攻击
  • 潜在伪造

2.7 证书

公钥证书

里面记有姓名，组织，邮箱，地址等个人信息，以及属于此人的公钥，并且由认证机构来施加数字签名（ 机构的私钥对发送者的私钥进行加密 ）

2.8 伪随机数生成器

2.8.1 随机数的性质

  • 1.随机性
  • 2.不可预测性
  • 3.不可重现性

2.8.2 随机数的分类

    弱伪随机数
        只具备随机性

    强伪随机数
        具备不可预测性

    真随机数
        具备不可重现性

3. 密钥

密码的本质就是把较长的秘密（消息）变成较短的秘密（密钥）

3.1 概念

密钥就是一个巨大的数字

密钥空间的大小由密钥长度决定

3.2 分类

  • 对称密码的密钥和公钥密码的密钥
  • 消息认证码的密钥与数字签名的密钥
  • 用于确保机密性的密钥与用于认证的密钥

    会话密钥与主密钥

    会话密钥：每次通信只能使用一次的密钥
    主密钥：一直被重复使用的密钥

3.3 密钥的管理

  • 生成密钥

    随机数生成
    口令生成

    PEB

    口令由人脑记住
    盐：随机数

  • 配送密钥

    Diffie-Hellman密钥交换

    通信双方通过交换一些可以公开的信息就可以生成共享的秘密数字

  • 更新密钥

    当前密钥的散列值作为下一个密钥

  • 保存密钥

    把密钥先加密后保存

    KEK（用于加密密钥的密钥）

  • 作废密钥

4. 密码算法

4.1 流密码

对数据流进行连续处理

OFB模式：输出反馈模式

密码算法的输出会反馈到密码算法的输入中

CTR模式：计数器模式

通过把逐次累加的计数器进行加密来生成密码流

4.2 分组密码

每次只能处理一定长度的一块数据
对分组密码算法进行迭代时所用的方法:

ECB模式：电子密码本模式

把明文分组加密后的结果直接成为密文分组

弱点：可以在不破译密文的情况下操纵明文

CBC模式：密码分组链接模式

把前一个密文分组与当前明文分组的内容混合起来加密

弱点：分组错误后，密文无法解密（如缺失1bit）

CFB模式：密文反馈模式

前一个密文分组会被送到密码算法的输入端（与CBC相比，可以理解成加密以后在混合）
