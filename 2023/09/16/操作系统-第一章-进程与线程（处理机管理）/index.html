操作系统-第一章-进程与线程（处理机管理）

发表于2023-09-16|更新于2023-09-21|操作系统

操作系统对进程与线程的管理（处理机管理）

1. 进程与线程-基础

1.1 进程

1.1.1 概念

具有独立功能的程序在一个数据集合上运行的过程，它是系统进行资源分配和调度的一个独立单位。这里的资源指时间和空间。 

创建进程，实际是创建进程实体中的PCB。
撤销进程，实际是撤销PCB。
进程是动态的，进程实体是静态的。

PCB：进程控制块 process control block

PCB的结构

描述信息：进程号或进程标识号（PID）等。
控制信息
资源管理信息（最大部分）
CPU现场保护结构

进程实体：对进程的动态描述，包括：

PCB+相关程序段+该程序段对其进行操作的数据结构

或者是 

PCB + 程序段 + 数据段

1.1.2 组成

A. PCB

包含内容

进程描述信息

PID(process)
TID(thresh)

进程控制和管理信息
资源分配清单
处理机相关信息

B. 程序段
能被进程调度程序调度到CPU执行对的程序代码段。
C. 相关数据段
对程序加工的原始数据，或者中间结果。

1.1.3 特征

  • 动态性
  • 并发性
  • 独立性
  • 异步性

1.1.4 状态与切换

  • 状态

    运行态
    就绪态

    排成一个队列，称为就绪队列

    阻塞态

    根据阻塞原因的不同，排成多个阻塞队列
    由阻塞原语切换入阻塞态

    创建态

    创建PCB
    分配资源

    内存
    文件
    I/O设备
    CPU时间

    终止态

    删除PCB
    回收资源

  • 切换
    从进行态到阻塞态是主动的。
    从阻塞态到进行态是被动的。

1.1.5 进程间的通信

有三种方式

共享存储
消息传递
管道通信

  • 共享存储
    两个或多个进程存在一块可以直接访问的共享空间，通过对共享空间的读写，实现进程之间的信息交换。

  • 消息传递

    直接通信方式

    直接发给目标进程

    间接通信方式

    先发给中间实体，中间实体称为信箱，然后从信箱提取信息

  • 管道通信
    管道是一种特殊的文件。
    普通管道只允许单向通信，如果要实现双向通信，要定义两个管道。

1.2 线程

1.2.1 线程的概念

引入线程的目的： 减小程序在并发执行时所付出的时空开销，提高操作系统的并发性能。

线程是基本的CPU执行单元。

1.2.2 线程与进程的区别

进程是除CPU外系统资源的分配单元。
线程是处理机的分配单元，CPU的基本执行单元。

1.2.3 一些术语

  • 特权指令/指令

    特权指令：

    有些指令是非常危险的，如果错用，将导致系统崩溃，比如清内存、设置时钟等，这些就是特权指令，不能随意使用。

    非特权指令：

    不会造成危险的指令。

    Intel 的 CPU 将特权等级分为 4 个级别：Ring0~Ring3。Linux 系统只使用了 Ring0 和 Ring3 两个运行级别(Windows 系统也是一样的)。
  • 内核态/用户态

    内核态

    进程运行在 Ring0 级别时被称为运行在内核态。

    用户态

    进程运行在 Ring3 级别时被称为运行在用户态。

  • 内核空间/用户空间

    内核空间

    在内核态下，进程运行在内核地址空间中，CPU可以执行任何指令。运行的代码也不受任何的限制，可以自由地访问任何有效地址，也可以直接进行端口的访问。

    用户空间

    在用户态下，进程运行在用户地址空间中，被执行的代码要受到CPU的诸多检查，它们只能访问映射其地址空间的页表项中规定的在用户态下可访问页面的虚拟地址，且只能对规定的可访问端口进行直接访问。

    可以这么理解，用户空间就是因为很安全所以可以“自由”，内核空间就是为了保证安全所以“限制自由”。

1.2.4 线程的实现方式

  • 用户级线程(ULT)
    有关线程管理的所有工作均在用户空间完成，内核意识不到线程的存在。

      优点
      1.不用状态切换，节省开销。
      2.线程的实现属于用户程序的一部分，与操作系统无关。

  缺点
      1.不能发挥多处理机优势。
      2.容易阻塞。

  • 内核级线程(KLT)
    线程管理的工作都在内核空间中实现，内核空间也为每个内核级线程设置一个线程控制块，内核通过内核控制块去控制线程。

    优点
    1.不阻塞。
    2.可以多处理器。

    缺点
    1.需要切换状态。

  • 组合方式

1.2.5 多线程模型

  • 1.一对一
    把一个用户级线程映射到一个内核级线程。

    优点

    不阻塞

    缺点

    空间开销大

  • 2.多对一

    优点

    效率高

    缺点

    1.阻塞
    2.多个线程无法同时在处理机上运行

  • 3.多对多
    将n个用户线程映射到m个内核级线程上。

    优点

    两者结合

2. 处理机调度

2.1 概念

进程数量往往多于处理机数量，因此进程会争用处理机。

对处理机进行分配，从就绪队列中按照一定的算法（高效，公平的原则）选择一个进程并分配处理机运行，以实现进程并发的执行。

2.2 调度的层次

  • 1.高级调度

    作业调度

    实质：内存与辅存之间的调度，每个作业只调入调出一次

  • 2.中级调度

    内存调度

    实质：把暂时不能运行的进程调至外存等待，此时进程的状态称为挂起态

    实际上是，存储器管理中的对换功能

  • 3.低级调度

    进程调度

    实质：分配进程

    调换频率很高

2.3 调度的评价标准

  • 1.CPU利用率

  • 2.系统吞吐量

  • 3.周转时间

  • 4.等待时间

  • 5.响应时间

2.4 调度的实现

2.4.1 调度程序（调度器）

  • 排队器
    把就绪进程排成队列。

  • 分派器
    把进程从就绪队列中取出，把CPU分配给该进程。

  • 上下文切换器

2.4.2 进程调度方式

  • 抢占式调度方式

  • 非抢占式调度方式
    适用于批处理系统
    不适用于分时系统和大多数的实时系统

2.4.3 典型的调度算法

  • FCFS
    First come first serve
    先来先服务。

  • SJF
    Short job first
    短作业优先算法。

  • 优先级调度算法
    非抢占式

    让正在处理的低优先级的进程先处理完

      抢占式

    关于优先级

    静态优先级

    优先级在整个运行时间保持不变

    动态优先级

    优先级会发生改变

  • 优先级的设置
    1.系统进程>用户进程

    2.交互型进程>非交互型进程

    3.I/O进程>计算型进程

    4.I/O处理慢，优先处理，提高整体效率

  • 高响应比优先调度算法
    选择相应比最高的作业来运行 

    响应比=（等待时间+要求服务时间）/要求服务时间

  • 时间片轮转算法
    采用FCFS算法的队列，不过每个进程只执行一个时间片

  • 多级队列调度算法

  • 多级反馈队列调度算法

3.同步与互斥

3.1 同步

进程的运行具有异步性，为了让进程与进程之间的运行有一些相互制约关系，采用进程同步，也称为直接制约关系。 

例如，挤牙膏和刷牙可以看成了两道程序，挤牙膏必须在刷牙前面，在计算机上执行时为了保证先后秩序，就要引入一种制约关系，就是同步。

3.2 互斥

称为间接制约关系。

对于临界资源，同一时刻不能有多个进程访问，引入互斥。    
例如，手机摄像头不可以一边扫码一边拍照。

3.5.1 概念

多个进程由于相互竞争资源造成的僵局。

3.5.2 产生的必要条件

  • 互斥条件

  • 不剥夺条件

  • 请求并保持条件

  • 循环等待条件

3.5.3 处理策略

1.死锁预防

破坏4个必要条件中的任意条件

2.避免死锁

防止系统进入不安全状态
不安全状态不一定死锁，死锁一定不安全状态
