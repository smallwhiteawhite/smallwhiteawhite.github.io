编译原理-第一章-引论

发表于2023-09-24|更新于2023-09-24|编译原理

引论

1. 程序设计语言

1.1 机器语言

01组成的序列。

1.2 汇编语言

引入了助记符这个概念，来表示指令中的操作和操作数。

  • 缺陷：
    如果单单使用助记符，必然受到具体机型的限制。

1.3 高级语言

高级语言不受具体机型的限制。

2. 程序设计语言的翻译

  • 编译程序
    把源程序翻译成汇编语言、机器语言程序，然后再处理、执行。

    eg.C,C++.

  • 解释程序
    一边读取源程序中的语句和输入的数据，一边执行；根据执行结果读取下一条语句，并
    再次解释并执行。
    eg.Python.

  • 优缺点
    解释程序人机交互性更好，但是效率低下。

3. 编译程序的总体结构


图片来源：https://pic1.zhimg.com/v2-a01ff9f95fdc5d595513982c1a119ee8_r.jpg
侵删。

3.1 词法分析

词法分析器完成。
输入：源程序的字符串。
输出：普通的字符流。

从左往右扫描组成源程序的字符串，并将其转化成单词串，将发现的
标识符登记到符号表中，检查词组错误并处理。

总结一下其功能

  • 1.把字符串转化成字符流
  • 2.检测标识符

    何为标识符：

    eg: int a = 10; a 便是一个标识符。

  • 3.词组分析

输出的格式是：二元组
（种别码，属性值）
eg:(标识符，a),(加号,+)

3.2 语法分析

语法分析器完成。
输入：词法分析器的输出，单词序列。
输出：单词组成的语句结构，比如语法树。

3.3 语义分析

完成语法分析程序识别出来的语法程序的语义的分析。

  • 语法制导翻译
    在语法分析器分析出语法成分的同时进行对语法成分的语义分析。

3.4 中间代码生成

以中间代码的形式实现对语义分析结果的表示。

机器语言和汇编语言极度依赖机器，用来直接用来表示语义分析的结果很不合适。

常见的中间代码表示：

  • 1.波兰式(后缀式)
  • 2.逆波兰式(前缀式)
  • 3.三元式
  • 4.四元式
  • 5.树式

3.5 代码优化

对中间代码进行优化，使程序尽量节省存储空间，更有效的利用机器资源。

代码优化分为全局优化和局部优化。

3.6 目标代码生成

完成从中间代码到目标机器上的机器指令代码或者目标机器上的汇编代码的转换。

对运算对象分配存储单元、寄存器等，把中间代码分配成等价的指令序列。

3.7 出错处理

  • 错误局部化
    把错误的影响范围降低，限制在局部。
  • 续编译
    错误局部化后，对剩余部分继续编译。

3.8 符号表管理

管理遇到的一系列的符号。
变量，常量，函数名字，标号等。

4. 编译程序的组织

  • 前端
    前端的工作是与源语言有关而与机器无关。

  • 后端
    后端的工作与机器有关。

划分前端和后端有类似封装的思想。
复用性和可移植性强。

5. 编译程序的生成

通过移植、自展完成新编译程序的构建。

5.1 自展(bootstrapping)

要编写一个编译高级语言的编译器，从最低级的语言开始不断扩展，最后到目标的高级语言位置。

想用A编译S，可以把S分成S1和S2两个子语言，S1和S2也可以分成S11、S21等子语言，知道划分到
可以用A编译的子语言为止。

不能一蹴而就时，采用步步为营的策略。

5.2 移植

P语言能在A和B两种机器上使用，用A机器上的语言AC去实现B机器上的语言BC。
