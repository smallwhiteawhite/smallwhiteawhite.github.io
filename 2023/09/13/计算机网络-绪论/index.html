计算机网络-绪论

发表于2023-09-13|更新于2023-09-17|计算机网络

计算机网络概述

1. 概念

计算机网络就是互联的、自治的计算机系统的集合

2. 计算机网络的组成

2.1 从组成部分上

计算机网络 = 硬件 + 软件 + 协议
  • 硬件： 主机，链路 and so on。
  • 软件： 就是各类软件。
  • 协议：计算机网络的核心，可以近似理解成一种规则、算法。

2.2 从工作方式上

计算机网络 = 边缘部分 + 核心部分
  • 边缘部分：主要是主机之间的连接，实现通信与数据共享
  • 核心部分：网络+连接网络的路由器，为边缘部分提供连通和交换服务

2.3 从功能组成上

计算机网络 = 通讯子网 + 资源子网
  • 通信子网：实现通信
  • 资源子网：实现资源共享

3. 计算机网络的功能

功能

1.数据通信
2.资源共享
3.分布式处理：避免一个网络中很多空闲一个过忙的情形
4.提高可靠性：网络中的计算机互为替代机
5.负载均衡：和分布式处理有点像

4. 计算机网络的分类

4.1 从分布范围上

  1. 广域网
  2. 城域网
  3. 局域网
  4. 个人区域网

4.2 从传输技术上

  1. 广播式网络
    所有联网的计算机共享一个公共信道，一个发报文全部都能”听到“报文。
  2. 点对点网络
    每条链路连接两台计算机；如果没有直接连接，采用中间节点存储、转发。

4.3 从拓扑结构上（要注意物理拓扑不等于逻辑拓扑）

1.星形
2.总线形
3.环形
4.网状

5. 计算机网络性能指标的衡量

  1. 带宽
    数字信道能传送的”最高数据传输速率”，单位bps（bit per second）。

  2. 时延
    数据从一段到另一端所用的时间。
    总时延 = 发送时延 + 传播时延 + 处理时延 + 排队时延（后两者一般可以忽略）。

  3. 时延带宽积

  4. 往返时间RTT

  5. 利用率
    包括网络利用率和信道利用率

  6. 吞吐量

6. 分层（很重要）

6.1 分层原则

独立
简洁
技术分割

可以类比成编程语言的封装思想，分层之后，每层只需要做好对上层的服务，对下层的接口，层内的规定自由设计。

6.2 分层方式

协议（水平的）
服务
接口（垂直的）

6.3 分层结构

6.3.1 7层OSI模型（法定标准）

应用层

表示能和用户交互产生网络流量的应用程序

表示层

交换信息的表示方式

例如，从1.jpg变成01编码形式

加密和解密
数据压缩恢复

会话层

向进程提供建立连接，并在其上有序的传送数据

传输层

负责两个进程之间的通信（端到端）
单位为报文段

网络层

把分组从源端传送到目的端
单位为数据报

链路层

把网络层传输的数据封装成帧
单位为帧

物理层

由电信号传输，在物理媒介上实现透明传输（透明指看不见）

6.3.2 4层TCP/IP模型（事实标准）

应用层
传输层
网际层
网络接口层

6.3.3 5层模型

应用层
传输层
网络层
链路层
物理层
