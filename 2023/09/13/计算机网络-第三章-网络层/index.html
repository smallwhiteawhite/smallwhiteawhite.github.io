计算机网络5层模型中的网络层

1. 功能

1.1 路由选择与转发（最佳路径）

路由算法

静态（非自适应路由算法）

管理员手工配置路由信息

动态（自适应）

路由器之间彼此交换信息，按路由算法优化出路由表项

全局性:链路状态路由算法OSPF
Open Shortest Path First，开放式最短路径优先协议，使用分布式的链路状态协议

特点:1.所有路由器掌握完整信息
2.适用于规模很大的互联网，且收敛速度很快
3.全部路由器到最后都建立一个链路状态数据库，即全网拓扑图
4.采用Dijkstra算法

分散性:距离向量路由算法RIP,Routing Information Protocol 一种分布式的，基于距离向量的路由选择协议，是因特网的标准协议

特点:1.只掌握相邻路由器的信息
2.利用跳数来作为计量标准，最多15跳
3.适用于小网络

路由协议

自治系统AS（方便管理路由器，同时保密路由协议）
在互联网中，一个自治系统(AS)是一个有权自主地决定在本系统中应采用各种路由协议的小型单位

内部网关IGP

一个AS内使用RIP,OSPF

外部网关EGP

AS之间使用BGP
BGP协议:边界网关协议：Border Gateway Protocol

BGP 是唯一一个用来处理像因特网大小的网络的协议，也是唯一能够妥善处理好不相关路由域间的多路连接的协议
特点:
1.支持CIDR
2.发生变化时，变更有变化部分

1.2 异构网络互联

1.3 拥塞控制

2.主要任务

传输分组从源端到目的端。

3.传输单位

数据报

IP数据报

首部 + 内容 = IP 数据报
首部 = 版本 + 首部长度 + 源地址 + 目标地址 + …

4.设备

路由器

  • 具备转发功能

  • 具备路由选择功能
    选择最恰当的，用时最短的路径，使用软件实现。

  • 由路由表生成转发表
    根据转发表进行转发

  • 路由器和网桥可以互联两个不同的网段，集线器不能。

5. 两大平面

网络层可以分为两大平面

  • 数据平面
    功能：完成转发
    地点：路由器内

  • 控制平面
    功能：完成路由选择
    地点：
    传统方法（每路由器法）：位于路由器内
    CDN方法：位于远程控制器内

6. 协议（重要）

6.1 IP协议

  • 名称：网际互连协议，Internet Protocol
  • 设计原则： 端到端。
    IP只为主机提供一种无连接、不可靠的、尽力而为的数据包传输服务。
  • 主要内容
    1.IP编址方案
    2.分组封装格式
    3.分组转发规则
  • 服务类型
    1.IP信息包的传送
    2.IP信息包的分割与重组

6.2 ARP

  • 名称：地址解析协议，即ARP（Address Resolution Protocol）
  • 功能
    根据IP地址获取物理地址。

6.3 ICMP

  • 名称：ICMP（Internet Control Message Protocol）Internet控制报文协议。

  • 功能：在IP主机、路由器之间传递控制消息。

    控制消息：控制消息是指网络通不通、主机是否可达、路由是否可用等网络本身的消息。这些控制消息虽然并不传输用户数据，但是对于用户数据的传递起着重要的作用。

    主要用于在主机与路由器之间传递控制信息，包括报告错误、交换受限控制和状态信息等。当遇到IP数据无法访问目标、IP路由器无法按当前的传输速率转发数据包等情况时，会自动发送ICMP消息。

6.4 IGMP

  • 名称：网际组管理协议。

  • 功能：运行在主机和组播路由器之间。

7.IP地址

7.1 IPv4

  • 4类IP地址
    A类：第一个字节为网络号，后三个字节为主机号。地址的网络号取值:1to126。
    B类：前两个字节为网络号，后两个字节为主机号。地址的网络号取值:128to191。
    C类：前三个字节为网络号，最后一个字节为主机号。地址的网络号取值:192to223。
    D类：是多播地址，D类IP地址的最前面为“1110”。地址的网络号取值:224to239。
    E类：是保留地址。E类IP地址的最前面为“11110”。地址的网络号取值:240to255。

  • IP地址
    =｛<网络号>，<主机号>｝
    全世界位唯一的32位标识符，标识路由器主机的接口。

  • IPv4地址紧张应对措施

    NAT：网络地址转换

    在专用网连接到因特网的路由器上安装NAT软件，安装NAT路由器，它至少有一个有效的外部全球IP地址。

    无分类编址CIDR

    消除传统的ABC类地址以及划分子网的概念，IP地址后加入“/”，然后写上网络前缀（可以任意长度）的位数。

7.2 IPv6

  • 基本地址类型

    单播

    一对一通信，可做源地址+目的地址

    多播

    一对多通信，可做目的地址
    三种常用算法

    基于链路状态的路由选择
    基于距离向量的路由选择

    任播

    一对多中的一个通信，可做目的地址

    128位地址
